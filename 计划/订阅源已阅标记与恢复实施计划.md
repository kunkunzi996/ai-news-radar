# 施工说明：订阅源卡片支持「已阅」标记，标记后移入「已阅」栏目、可随时恢复

> 给 Codex 的任务说明。请严格按本文件执行，不要自由发挥，不要顺手改其它无关代码。
> 项目根目录：`E:\AI-news-reader\ai-news-radar-run`（注意不是 `E:\AI-news-reader`，那是外层容器目录）
> 当前分支：`master`（继续在此分支改即可，**不要**新建分支、不要 commit / push，除非用户另行授权）

> 背景：看板是个汇总工具，"我的订阅"栏目下的各路信源（B站动态、抖音博主、小红书博主、微信公众号、GitHub Release、YouTube 频道……）每次刷新都会把近期内容重新拉一遍。用户经常已经在对应平台官网看过某条内容了（不只是 B 站，抖音/小红书/公众号等同理），但只要它还在采集窗口内，看板就会一直显示，妨碍阅读。用户想要一个"已阅"操作：点了就从主看板消失，进到一个专门的"已阅"栏目里存着；如果点错了，能从"已阅"栏目一键"恢复"回来。**这个能力对"我的订阅"下的所有信源类型都适用，不局限于 B 站**（B 站只是用户描述需求时举的例子）。
>
> 技术选型：这个项目是纯静态站点（`data/*.json` 由 Python 脚本生成，前端 `assets/app.js` 只读不写这些文件），"已读"这种个人标记不适合写回会被下次采集整体覆盖的 `data/*.json`。项目里已经有先例——信源配置草稿（`SOURCE_CONFIG_STORAGE_KEY`）、采集范围选择（`COLLECTION_SCOPE_STORAGE_KEY`）都是存在浏览器 `localStorage` 里的。本次"已阅"状态照这个先例，同样只存 `localStorage`，不改后端、不改 `data/*.json`、不改 `scripts/local_server.py`。只要这条记录还留在 `data/latest-24h-all.json` 的 `items_all` / `items_all_raw` / `creator_items_all` 数组里（没有被信源删除联动清理掉），"已阅"标记就能正确生效并在刷新后继续隐藏它。

## 一、目标（要做成什么样）

做完之后：

1. 在"我的订阅"及其下属的"抖音""小红书""微信公众号""B站""油管"等栏目里，每条**订阅源卡片**（B 站动态、抖音博主、小红书博主、微信公众号、GitHub Release、YouTube 频道——凡是会出现在"我的订阅"栏目里的内容，不限平台）右下角都会出现一个"已阅"按钮。点一下，这条卡片立刻从当前所有栏目/列表里消失（不管用户在哪个栏目、有没有搜索/筛选，只要标记过已阅就不再出现在常规视图里）。

3. 在"已阅"栏目里，同一张卡片的按钮变成"恢复"。点一下，这条卡片从"已阅"栏目消失，回到它原本所在的"我的订阅"/对应平台栏目里能重新看到。

**本次范围做「我的订阅」栏目下所有信源类型的已阅/恢复，不做以下事：**
- 不给"我的订阅"之外的普通信息流卡片（官方一手源、精选媒体、AI HOT、聚合站点、研究论文、社区帖子等非订阅类内容）加"已阅"按钮——这些内容不是"你关注的具体账号"，不适用"已经在原平台看过"这个场景，本次不碰。
- 不改后端：不碰 `scripts/local_server.py`、`scripts/update_news.py`、`data/*.json`——已阅状态纯前端存储，不写回任何数据文件。
- 不做"已阅"状态跨浏览器/跨设备同步（`localStorage` 天然按浏览器隔离，本次不引入账号体系或服务端存储去解决这个问题）。
- 不做"已阅列表自动清理/过期"机制（如果某条记录未来被信源删除联动清理出了 `data/*.json`，这条已阅标记会变成找不到对应内容的孤儿记录，静默无效，不影响其它功能，本次不专门处理这种小概率情况）。

function loadReadItemIds() {
  try {
    const raw = window.localStorage.getItem(READ_ITEMS_STORAGE_KEY);
    const arr = raw ? JSON.parse(raw) : [];
    return new Set(Array.isArray(arr) ? arr : []);
  } catch {
    return new Set();
  }
}

function persistReadItemIds() {
  try {
    window.localStorage.setItem(READ_ITEMS_STORAGE_KEY, JSON.stringify(Array.from(state.readItemIds)));
  } catch {
    // localStorage 不可用（隐私模式/容量满），已阅状态本次会话不持久化，不影响当前操作。
  }
}

function isItemRead(item) {
  const key = readTrackingKey(item);
  return Boolean(key) && state.readItemIds.has(key);
}

function toggleItemRead(item) {
  const key = readTrackingKey(item);
  if (!key) return;
  if (state.readItemIds.has(key)) {
    state.readItemIds.delete(key);
  } else {
    state.readItemIds.add(key);
  }
  persistReadItemIds();
  rerenderCurrentView();
}

function isSubscriptionItem(item) {

## 八、红线（务必遵守）

- 不要新建分支、不要 commit / push（用户验收后自己决定）。
- 不要改本说明清单之外的文件，尤其**不要碰 `scripts/local_server.py`、`scripts/update_news.py` 或 `data/` 目录下任何文件**——已阅状态本次只存浏览器 `localStorage`，跟后端和数据文件完全无关。
- 不要做批量文件删除操作。
- **"已阅"按钮只加在订阅源卡片上**（判断条件必须是 `isSubscriptionItem(item)`），不要放宽到"我的订阅"栏目之外的普通信息流卡片（官方源、聚合站点、研究论文等）。
- **已阅/排除已阅的过滤逻辑只加在 `sectionItems` 这一个函数里**，不要在 `getFilteredItems`、`renderSectionTabs`、`currentSiteStats` 等下游函数里重复添加类似判断——这些函数都依赖 `sectionItems` 的结果，重复添加容易造成逻辑分叉、后续维护困难。
- `loadReadItemIds` / `persistReadItemIds` 里的 `localStorage` 读写**必须包在 try/catch 里**，静默降级（读失败当空集合处理，写失败不报错、不阻断当前这次标记/恢复操作），不能因为 localStorage 不可用（隐私模式、容量满）而导致页面报错或白屏。
